# Phase 06 – Final Report

## Overview

This project simulated a security incident within a controlled lab environment using a SIEM platform.

The objective was to demonstrate how security events are collected, detected, analyzed, and handled through a structured incident response process.

---

## Lab Summary

The environment consisted of:

* A SIEM platform for monitoring and analysis
* A monitored endpoint generating logs
* A simulated attacker scenario

---

## Incident Summary

A sequence of suspicious activities was detected on the monitored system, including:

* Multiple failed attempts to execute privileged commands
* Successful privilege escalation to root
* Creation of a new unauthorized user account

These actions indicate a potential compromise of a legitimate user account.

---

## Key Findings

* The SIEM effectively detected suspicious activity in real time
* Privilege escalation attempts were identified through alert correlation
* Persistence techniques (user creation) were successfully logged and analyzed

---

## Lessons Learned

* Continuous monitoring is essential for detection of threats
* Privileged access should be strictly controlled and monitored
* Correlation of multiple events is key to identifying real security incidents

---

## Conclusion

This project demonstrates my practical skills in SIEM usage, incident detection, and analysis.

It highlights the importance of proactive monitoring and structured incident response in a modern cybersecurity environments.
